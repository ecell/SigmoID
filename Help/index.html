SigmoID overview

SigmoID front end is a GUI application written in Xojo which gives it the expected look and usability on all three supported platforms (OS X, Linux and Windows). The GUI provides the interface for selected programmes from the HMMER, MEME Suite and TransTerm HP, which are responsible for the actual searches. Sequence logos are calculated by WebLogo which is written in Python, hence Python and BioPython (version 1.64 and above) are required. Processing of nhmmer, mast and TransTerm HP output, sequence format conversions and adding regulatory sites to genome annotation are implemented as separate Python scripts. The scripts are called from the GUI, but could easily be used separately and integrated in an annotation pipeline if desired. Detailed installation instructions are provided within distributions for each platform. The source code for the whole SigmoID application is available with GPL 2.0 license.

Functions:
- visualise binding site alignments with WebLogo;
- extend, shorten and mask alignments;
- create hmm profiles from alignments;
- search bacterial genomes with calibrated (and uncalibrated) hmm profiles;
- annotate promoters and transcription factor binding sites in GenBank-formatted genomes.

This version includes 36 calibrated profiles (for 5 sigma factors and 31 TFs) optimised for enterobacterial phytopathogens Pectobacterium spp. and Dickeya dadantii. Efficiency of these profiles may decrease for other bacteria.

The search for binding sites is done by nhmmer which is expected to be installed in the default location. WebLogo installation is required for new alignments, otherwise logos won't be displayed, but the rest of SigmoID should work.

Adding annotations to GenBank files is done by the HmmGen.py script which could be used on its own. BioPython (version 1.64 and up) is required.

At the moment SigmoID is known to work on OS X (10.8-10.11), Windows (7 and 8), Ubuntu (12.04 and 14.04). It may also work on other Linux distributions, provided the required libraries are installed. 64 bit Linux distibutions require installation of 32-bit libraries.

Optimising profiles

The binding site sequences extracted from RegPrecise (or from original publications) were used to build a primary hmm profile and search for the similar sites in genomes of at least four different species, usually Pectobacterium carotovorum, P. atrosepticum, P. wasabiae and Dickeya dadantii. Orthologous binding sites found by nhmmer were always retained. Contrary to the conservative approach utilised by RegPrecise, additional high scoring sites were considered and retained if located in front of the genes known or likely to be involved in the same physiological process as the characterised genes of the regulog. This resulted in secondary alignment of binding sites composed exclusively of Pectobacterium/Dickeya sequences. At this point we evaluated the possibility to improve signal/noise ratio by verifying the boundaries of the binding site alignments. To do this, we applied the "Extend binding sites" function of SigmoID, adding ten bases at each side of binding sites, verified the presence of conserved bases in extended regions and then shrinked the extended sites back to the conserved bases.

Finding profile thresholds

Saving calibrated profiles

Utility functions are provided for extension/shortening/partial masking of an existing alignment of binding site sequences. Displaying the logo of the currently loaded binding site alignment helps to choose the correct boundaries of a binding site. Genome browser also gives an opportunity to display nhmmer hits in the genomic context, where the hits could be selected or deselected and the currently selected set of binding sites could be saved in fasta format. Finally, a calibrated hmm profile could be saved. Currently calibration thresholds have to be provided by the user, but choosing the right calibration values is simplified by looking through nhmmer hits in the genome browser.


Searching for regulatory sites in a genome

SigmoID can search for transcription/sigma factor binding sites and for terminators. Terminator search can be launched at any time, while binding site searches require loading binding site data first. Once a binding site data is loaded, either nhmmer or MAST could be used to search a genome for the corresponding binding sites. A genome browser window that opens after the completion of the search allows to quickly skim through the binding sites just found, inspect the neighbouring genes and correct their annotation if desired. The sites found during the last search could be selected/deselected and the selection could be saved as a fasta file. If a set of pre-calibrated profiles is available, the Genome Scan function could be used to search for automatic search for all binding sites and terminators in a given genome.

Database searches and editing annotation

Genome browser is opened after a search for binding sites and could be used to quickly skim through the sites just found. Alternatively, the browser could be used to view an existing GenBank file independently of any search function. SigmoID allows basic editing or deleting of existing features and their qualifiers and provides conveniently integrated web-based hmmersearch, hmmscan, blastn and blastp search functions for verifying the existing annotations. The search is possible against protein, nucleotide and domain databases. The access to these functions is provided via the corresponding APIs (12, 13). The browser can also be used to search for a sequence, coordinate or feature text within the current genome.

Genome scan

We provide pre-calibrated hmm profiles for 26 transcription regulators optimised for Pectobacterium and Dickeya genera. The Genome Scan function will search a chosen genome for sites for each of these profiles (or profiles created by the user) and transcription terminators and add corresponding annotations to the GenBank file.
